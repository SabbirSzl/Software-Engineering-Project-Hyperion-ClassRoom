1. Introduction

1.1 Purpose

The purpose of this document is to present a detailed description of the classroom software Hyperion. It will explain the purpose and features of the software, the interfaces of the software, what the software will do and the constraints under which it must operate. This document is intended for users of the software and also potential developers.

1.2 Document Conventions

This document has been created based on the IEEE template for System Design Specification (SDS) documents.

1.3 Intended Audience

  • Developers, who will be developing this software.
  • Teachers, who are among the primary users of this software.
  • Students, who will be the most beneficiary of this software.
  • Programmers, who are interested in working on the project by further developing it or fix existing bugs.

1.4 Product Scope

Hyperion is a virtual platform which simplifies the interactions between students and teachers by creating a classroom environment. Due to the COVID-19 global pandemic, the demand for online interaction platforms for students and teachers has grown. Software like Google Classroom already offer such platforms but our aim is to offer what they lack and make life easier especially for teachers and invigilators by providing them with tools which will aid them in ensuring fair grading.

2. Description of Design Components

A component provides a particular function or group of related functions. In our Hyperion Classroom Software System we have few components like classroom component, user component etc.

  • User Component : This is one of the key component of our Hyperion classroom Software System. User`s acts as a Teacher or Student. If a user creates a class then automatically that user assigned as Teacher of that class.Teacher can add other user as Student or Student can join in particular classes.
  • Classroom Component : This component is responsible for the classes user have created or joined. Classroom interfaces will be different for Teachers and student. Classroom component is responsible for opening particular class.
  • Authentication and User management component : This is the major sub system that is responsible for the security of the Hyperion Classroom Software System. It authenticate users and also handles user management activities such as log in to the system.
  • Post Creation Component : This component is responsible for creating post in classroom. Both User and Teacher can create a post to the classroom. and both can comment at the post.
  • Task Assign Component : This component is responsible for assigning task to the Classroom. System will publish the task for all the students assigned in that particular class. Student will submit the task and teacher will grade the submission.
  • Uploading File Component : User can upload files such as pdf or word in the classroom system. Uploaded files will be available for eveyone in that classroom. This component is responsible for uploading a file and publish for the users of the classroom.
  • Report Generation Component : This is a relatively small subsystem compared to the other components of the Hyperion Classroom Software System. This is the component which is responsible for generating report of students like how much time they spend on particular file.

    • 3. High-Level Component Design

    Singleton Pattern

    This is a creational design pattern and is one of the simpest pattern in the field of software engineering. It involves only one class which is responsible to instantiate itself,so that it creates no more than one instance. The singleton pattern is useful when access to limited resource needs to be controlled.
    Applications:
    In the Hyperion classroom software system this pattern will be use for viewing documents or uploaded file. So that system can generate a distinct report for a single user.

    Observer Pattern

    The observer pattern is a behavioral pattern which defines a one-to-many dependency between objects where a state change in one object results with all its dependents being notified and updated automatically. This pattern may be used in all situations where more than one display format for state information is required and where it is not necessary for the object that maintains the state information to know about the specific display format used.
    Applications:
    Observer pattern will be used in the Hyperion classroom software system for the operations of the system users.

    Adapter Pattern

    This adapter pattern is a structural pattern that translates one interface for a class to a compatible interface. This will convert the interface of a class into another interface that the user expects. Adapter gives the opportunity for the classes with incompatible interfaces to work together.
    Applications:
    This pattern will be used in the Hyperion classroom software system when displaying information from the database.

    Template Pattern

    In template pattern, an abstract class exposes defined ways to execute its methods. Its subclasses can override the method implementation as per need but the invocation is to be in the same way as defined by an abstract class. This pattern comes under behavior pattern category.
    Applications:
    In the Hyperion classroom software this pattern will be used to provide template for controller with some predefine methods.

    Strategy Pattern

    In strategy pattern,we create objects which represent various strategies and a context object whose behaviour varies as per its strategy object. The strategy object changes the executing algorithm of the context object.
    Applications:
    In the Hyperion classroom software system we will use this pattern to document conversion. We will change the type of document into a specific type.

    6. Database Design

    6.1 ER Diagram(s)

    6.2 Table Description(s)

    Table Name: user
    Purpose: Store general user information and credentials.

    Attribute Name Description
    ID Store a unique user ID that will help identify a particular user.
    First_name Store user First Name
    Last_name Store user Last Name
    DOB Store user Date of Birth
    Email Store a unique user Email
    Registration_date Store the date of registration

    Table Name: user_login
    Purpose: Store user password

    Attribute Name Description
    ID Store user ID that refers to ID column of user table
    Password Store hashed user given password.

    Table Name: teacher
    Purpose: Store teacher user information

    Attribute Name Description
    ID Teacher user ID that refers to ID column of user table
    Classroom_Id This column stores the classroom ID where the user is assigned as teacher. It refers to the ID column of classroom table

    Attribute Name: student
    Purpose: Store student user information

    Attribute Name Description
    ID Student user ID that refers to ID column of user table
    Classroom_Id This column stores the classroom ID where the user is enrolled as student. It refers to the ID column of classroom table

    Table Name: classroom
    Purpose: Store classroom related information

    Attribute Name Description
    ID Store unique classroom ID
    Title Store classroom title or name for a particular classroom
    Description Store classroom description
    Creator_Id Stores the user ID of the user who created the classroom
    Creation_Date Store the classroom creation date
    Classroom_Code Store unique classroom invitation code that can be used to join the class

    Table Name: task
    Purpose: Store classroom assigned task related information

    Attribute Name Description
    ID Store unique task ID
    Title Store classroom task title
    Description Store task description
    Teacher_ID Store teacher user ID who assigned the task. It refers to the ID column of user table
    Classroom_Id Store classroom ID where the task is posted that refers to the ID column of classroom table
    Content_ID Store the content ID attached in the task that refers to the ID column of content table
    TimeStamp Store the date and time of the assignment of task

    Table Name: post
    Purpose: Store classroom post related information

    Attribute Name Description
    ID Store unique post ID
    Title Store the title of the post
    Classroom_Id Store the classroom ID that refers to the column ID of classroom table
    User_ID Store the ID of user who posted. It refers to the column ID of user table
    Timestamp Store the date and time when the post was posted
    Description Store the description that contains in the post body
    Content_ID Store the content ID that refers to the column ID of content table
    LastModified Store the time and date of the last modification time

    Table Name: content
    Purpose: Store attachment content related information

    Attribute Name Description
    ID Store the unique content ID
    User_ID Store the user ID who posted the attachment that refers to the ID column of user table
    Classroom_ID Store the ID of classroom where content needs to be attached. It refers to the ID column of classroom table
    Attachment Store the attachment files as binary file
    TimeStamp Store the date and time of when the content was attached

    Table Name: comment
    Purpose: Store classroom post comment related information

    Attribute Name Description
    Classroom_ID Store the ID of classroom where comment needs to be posted. It refers to the ID column of classroom table
    Post_ID It refers to the ID column of post table. It stores the post ID where the comment needs to be attached
    TimeStamp Store the time and date of comment
    LastModified Store the date and time of last modification of the comment
    Content Store the comment body

    Table Name: submission
    Purpose: Store submission related inforamtion

    Attribute Name Description
    ID Store unique submission ID
    User_ID Store the user ID who made the submission. It refers to the ID column of user table
    TimeStamp Store the date and time of the submission
    LastModified Store the last modification time and date
    Content_ID Store the content ID of attachment that refers to the ID column of content ID
    Classroom_ID Store the classroom ID where the submission is made. It referes to the ID column of classroom table

    Table Name: analytic
    Purpose: Store the analytic report related information

    Attribute Name Description
    ID Store unique analytic ID
    Classroom_Id Store the classroom ID where the report is generated. It refers to the ID column of classroom table
    Teacher_Id Store the teacher user ID who requested the analytic report. It refers to the ID column of user ID
    Generation_TimeStamp Store the date and time of report generation
    Report Store the report files as binary

    Table Name: admin
    Purpose: Store admin information and credentials

    Attribute Name Description
    ID Store a unique admin ID that will help identify a particular admin.
    First_Name Store the first name of the admin
    Last_Name Store the last name of the admin
    DOB Store the date of birth of admin
    Email Store the admin email

    Table Name: admin_login
    Purpose: Store admin password

    Attribute Name Description
    ID Store admin ID. It refers to the ID column of admin table
    Password Store hashed admin password
